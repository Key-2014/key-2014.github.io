## Introduction

When you want to try out a personal project or a quick Web app idea, having a workflow that goes **Create repository from a template instantly** → **Develop locally** → **Auto-deploy to GitHub Pages** is incredibly convenient.

This article uses the [**web_app_test**](https://github.com/Key-2014/web_app_test) repository (a scientific calculator app) that I actually created as an example to explain the entire process from creating a Web app development repository to deploying it to GitHub Pages.

## 2. Creating a New Repository

### 2.1 Initializing the Project with Vite

First, use `create-vite` to generate a React + TypeScript template.

```bash
# If pnpm is not installed
npm install -g pnpm

# Create a project (e.g., named my_web_app)
pnpm create vite my_web_app --template react-ts
```

This sets up a complete React + TypeScript development environment.

### 2.2 Creating a GitHub Repository

1. Create a new repository on [GitHub](https://github.com) (e.g., `my_web_app`).
2. Initialize your local repository and push:

```bash
cd my_web_app
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/<username>/my_web_app.git
git push -u origin main
```

## 3. Directory Structure

The directory structure generated by the Vite + React + TypeScript template is as follows:

```text
my_web_app/
├── .github/
│   └── workflows/
│       └── deploy.yml          # GitHub Pages deployment workflow
├── public/                     # Static assets (favicon, etc.)
├── src/
│   ├── assets/                 # Assets like images
│   ├── App.tsx                 # Main React component
│   ├── main.tsx                # Entry point
│   └── index.css               # Global CSS
├── index.html                  # HTML template
├── package.json                # Dependencies and scripts
├── tsconfig.json               # TypeScript config (root)
├── tsconfig.app.json           # TypeScript config (app)
├── tsconfig.node.json          # TypeScript config (node)
├── vite.config.ts              # Vite configuration
├── eslint.config.js            # ESLint configuration
└── .gitignore                  # Git ignore file
```

### Key Files

- **`src/App.tsx`** — The main component defining the app's UI.
- **`src/main.tsx`** — The entry point that mounts the React app to the DOM.
- **`vite.config.ts`** — Vite build configuration (plugins, base path, etc.).
- **`index.html`** — The base HTML for the SPA. React is mounted to `<div id="root">`.

## 4. Introducing Tailwind CSS

In Tailwind CSS v4, it can be introduced as a Vite plugin.

### 4.1 Installation

```bash
pnpm add tailwindcss @tailwindcss/vite
```

### 4.2 Updating Vite Config

Add the Tailwind plugin to `vite.config.ts`:

```ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react-swc';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  plugins: [react(), tailwindcss()],
  base: '/my_web_app/',  // Base path for GitHub Pages
});
```

### 4.3 Importing in CSS

Add the following to `src/index.css`:

```css
@import "tailwindcss";
```

Now you can use Tailwind CSS. Just write Tailwind utility classes in `className`, and the styles will be applied.

## 5. Local Development

### 5.1 Installing Dependencies

```bash
pnpm install
```

### 5.2 Starting the Development Server

```bash
pnpm dev
```

Access `http://localhost:5173/my_web_app/` in your browser, and a development server with hot-reload support will start. The browser will automatically update every time you save your source code.

### 5.3 Other Commands

| Command | Description |
| --- | --- |
| `pnpm dev` | Starts the development server |
| `pnpm build` | Executes a production build (outputs to `dist/`) |
| `pnpm preview` | Previews the built app locally |
| `pnpm lint` | Statically analyzes the code with ESLint |

## 6. Deploying to GitHub Pages

Set up GitHub Actions to automatically build and deploy upon pushing to the `main` branch.

### 6.1 `base` Setting in `vite.config.ts`

For GitHub Pages, the repository name becomes a subpath, so you need to set the `base`:

```ts
export default defineConfig({
  plugins: [react(), tailwindcss()],
  base: '/my_web_app/',  // Match your repository name
});
```

### 6.2 Creating a GitHub Actions Workflow

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy static content to Pages

on:
  push:
    branches: ['main']
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: 'pages'
  cancel-in-progress: true

jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v5
      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: latest
      - name: Set up Node
        uses: actions/setup-node@v6
        with:
          node-version: lts/*
          cache: 'pnpm'
      - name: Install dependencies
        run: pnpm install --frozen-lockfile
      - name: Build
        run: pnpm run build
      - name: Setup Pages
        uses: actions/configure-pages@v5
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v4
        with:
          path: './dist'
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### 6.3 GitHub Repository Settings

1. Open your repository's **Settings → Pages**.
2. Set the **Source** to **GitHub Actions**.

Now, every time you push to the `main` branch, the build and deployment will run automatically.

## 7. Practical Example: web_app_test (Scientific Calculator)

As a practical example of a repository created using the above steps, I introduce [**web_app_test**](https://github.com/Key-2014/web_app_test).

### Overview

- **App Content**: A scientific calculator using Tailwind CSS v4.
- **Features**: Basic arithmetic, trigonometric functions (sin/cos/tan), logarithms (ln/log), square root, exponentiation, Degree/Radian switching.
- **Deployed at**: [https://key-2014.github.io/web_app_test/](https://key-2014.github.io/web_app_test/)

### Key Points

- A simple structure where all logic and UI are contained in `App.tsx`.
- Supports keyboard input (number keys, Enter to calculate, Backspace to delete, Escape to clear).
- Achieves fast builds using `@vitejs/plugin-react-swc`.

## Summary

To summarize the steps for creating a Web app development repository:

1. Initialize the project with `pnpm create vite`.
2. Introduce Tailwind CSS or other tools as needed.
3. Develop locally with `pnpm dev`.
4. Place `.github/workflows/deploy.yml`.
5. Change the Source to GitHub Actions in the GitHub Pages settings.
6. Push to the `main` branch for automatic deployment!

Using this workflow, you can quickly shape and publish your new Web app ideas. Please give it a try 🚀
